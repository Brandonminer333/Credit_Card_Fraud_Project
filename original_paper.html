Credit Card Fraud Modeling

Brandon Miner and Konnor Payne

Introduction

The focus of this paper was to create a regression model to accurately predict whether a transaction is fraudulent from parameters including time, location, etc. The models were built on the Credit Card Fraud Detection 2023 dataset obtained from Kaggle.com by Nidula Elgiriyewithana, which contains information on European credit cardholders. The dataset consists of 31 parameters in which 28 were anonymized to protect the credit card holders’ information; the 3 parameters that were made public are as follows: ID (unique identifier), Amount (transaction amount), and Class (binary fraud label).

Since we are both statisticians, we found this dataset of particular interest as it allowed us to build complex models to predict fraudulent transactions. Our first task was to clean the dataset and identify trends that could inform model selection.

A histogram of Amount was created, followed by a Principal Component Analysis (PCA) to reduce dimensionality and drop insignificant parameters. The processed data was then used to construct a series of regression-based models, optimized for accuracy, sparsity, and false negative rate.

Data Analysis

Before analysis, the data was cleaned and standardized. The ID parameter was removed as it held no predictive value. Histograms of fraudulent and non-fraudulent transactions (Figure 1) were analyzed to identify any patterns, but Amount appeared insignificant for classifying fraud.

A PCA using the Singular Value Decomposition (SVD) method was performed to identify the most informative components. Equations (1) and (2) standardized and decomposed the dataset respectively. Only the top principal components (PC1 and PC2) were retained, explaining 43.19% of total variance.

The scatter plot of transactions projected onto PC1 and PC2 (Figure 3) revealed distinct groupings between fraudulent and non-fraudulent transactions, particularly along PC1. Parameters most correlated with PC1 were deemed critical for fraud prediction.

A PCA Biplot (Figure 4) identified which parameters contributed most strongly to PC1. The parameters V1, V3, V5, V6, V7, V9, V10, V12, V14, V16, V17, and V18 were positively correlated with PC1; V2, V4, V11, and V19 were negatively correlated. These 16 total parameters formed the final model’s feature set.

Modeling

To reduce computational load and improve interpretability, model sparsity was prioritized. Sparse models exclude unimpactful parameters to reduce runtime and memory usage while maintaining predictive accuracy.

All models used an 80/20 train-test split and were based on linear regression. The linear regression formula (Equation 3) was used to predict binary outcomes, with outputs ≥ 0.5 classified as fraudulent.

Ridge regression (Equation 5) was implemented to improve time complexity via regularization, while LASSO regression (Equation 6) added a sparsity-promoting term controlled by λ. Increasing λ increases sparsity but can reduce accuracy.

Initial linear and ridge regression models achieved ~94.7% mean accuracy. LASSO regression maintained >90% accuracy with λ = 0.4, achieving an 11.2% false negative rate. Table 1 summarizes model performance.

Discussion

One of the earliest findings was the insignificance of Amount in predicting fraud. Contrary to initial assumptions, the data showed no correlation between transaction value and fraudulence. This might be due to existing fraud detection protocols that already scrutinize high-value transactions.

While anonymized parameters limit interpretability, the models remain useful for banks. Each can serve as a real-time fraud detection system where the fraud probability threshold can be adjusted to balance false negatives against computational efficiency.
